Rust'ın Süper Gücü: Ownership

Rust'ın bellek güvenliğinin sırrı üç basit kurala dayanır: her değerin tek bir sahibi vardır, aynı anda ya bir değiştirilebilir referans ya da birden fazla salt okunur referans olabilir, ve referanslar her zaman geçerli olmalıdır.

OTOMATİK ANİMASYON
OWNERSHIP ÖRNEĞİ
fn main() {
    let s1 = String::from("Sipahi"); // s1 sahibi
    let s2 = s1; // Ownership s2'ye taşındı (move)
    // println!("{}", s1); ← DERLEME HATASI! s1 artık geçersiz
    println!("{}", s2); // ✓ s2 geçerli tek sahip

    let s3 = s2.clone(); // Derin kopya — iki bağımsız sahip
    println!("{} {}", s2, s3); // ✓ İkisi de geçerli
}

C++'da bu kod çalışır — iki pointer aynı belleğe işaret eder, biri serbest bırakılınca diğeri "dangling pointer" olur. Rust'ta derleme zamanında yakalanır — program çalışmadan önce hata bulunur. Sıfır runtime maliyeti.

"C'de 10 yılda bulunacak bellek hatasını, Rust derlerken yakalar. Bu yüzden Sipahi çekirdeği Rust ile yazılıyor — çünkü hava savunma sisteminde 'segmentation fault' lüksümüz yok." 🦀

Rust'ın Yolculuğu

2006
Graydon Hoare kişisel proje olarak Rust'ı başlatıyor. İsim, rust mantarından geliyor.
2009
Mozilla projeyi resmi olarak desteklemeye başlıyor. Servo tarayıcı motoru için geliştiriliyor.
2015
Rust 1.0 yayınlanıyor. Stabilite garantisi başlıyor. Ownership sistemi olgunlaşıyor.
2016–2024
Stack Overflow anketlerinde 9 yıl üst üste "en sevilen dil" seçiliyor.
2022
Linux 6.1 Rust'ı resmi çekirdek dili olarak kabul ediyor. Tarihi bir an.
2024–2026
Sipahi Microkernel — RISC-V üzerinde Rust ile safety-critical RTOS geliştiriliyor.
